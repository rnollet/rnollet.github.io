Sandbox and personnal website
=============================


Comment faire un beau site web ?
--------------------------------

Le challenge étant de combiner :

- **Ne pas y passer trop de temps** Parce que ça n’est pas essentiel. Le but
  est que les gens y trouvent les infos qu’ils cherchent, pas qu’ils passent
  des heures à s’extasier devant.
- **Que ce soit simple** Que je ne me perde pas dans du code auquel je ne
  comprends plus rien ou dans une API obscure.
- **Que ce soit beau** Que ça fasse professionnel. Que ça n’ait pas l’air d’un
  vieux markdown tout moche.
- **Que ce soit portable** Que si je change d’hébergeur, ou si je veux le
  mettre sur la page du site de mon institution, etc. ce soit juste faisable.

La combinaison de tout ça semble entraîner que je veux du html, css, js un peu
compliqué, pour faire des trucs beaux, mais que je ne veux pas l’écrire
moi-même.

J’ai lu
- Hugo
- Wowchemy
- Academic
- Netlify
- Hugo is Academic’s underlying HTML generator
- Academic et Wowchemy sont des templates, des themes pour Hugo.
- Mais Wowchemy se présente comme “the website builder for Hugo”
- Wowchemy est le nouveau nom, nouvelle version, fusion de projets de Academic
- Wowchemy v5 prend la suite de Academic v4.8
- Il y avait deux “Academic” : le Academic template et le Academic
  ecosystem/framework, qui est plus généralement un website builder

- cette page de blog a l’air un peu vieille mais donne des infos intéressantes
  https://hermanp.github.io/en/post/how-this-blogdown-site-was-created/

- Jekyll and WordPress on the other hand have complex Ruby and PHP
  dependencies, respectively. (Contrairement à Hugo qui est assez simple et
  assew self-contained.) Jekyll took significantly longer and occasionally
  stopped working due to dependencies becoming deprecated or packages requiring
  to be manually updated.
